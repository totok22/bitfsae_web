# 常用命令备忘录

本文档整理了 FSAE 遥测项目服务器维护及数据调试过程中常用的命令。

- 宿主机 `nginx` 负责 `80/443`
- 宿主机 `pm2` 负责 Web 进程 `bitfsae`
- `docker compose` 只负责 `mosquitto`、`influxdb`、`telegraf`、`grafana`

## 1. 服务器日常维护

这些命令需在服务器的 `/home/admin/fsae_project` 目录下执行，除非特别说明。

```bash
# 进入项目目录
cd /home/admin/fsae_project

# 停止遥测与监控容器
sudo docker compose down

# 启动遥测与监控容器
sudo docker compose up -d

# 查看 Compose 服务运行状态
docker compose ps

# 重启特定服务
docker restart fsae_telegraf
sudo docker compose restart grafana

# 查看 Nginx 状态
systemctl status nginx --no-pager

# 查看 PM2 状态
pm2 ls
pm2 logs bitfsae
```

### 编辑配置文件

```bash
# 编辑 Telegraf 配置 (数据采集规则)
nano telegraf/telegraf.conf

# 编辑 Protobuf 定义 (数据结构)
nano /home/admin/fsae_project/protos/fsae_telemetry.proto

# 编辑 Docker 编排文件
nano /home/admin/fsae_project/docker-compose.yml
```


## 2. 数据链路调试

用于检查数据是否正常从 MQTT 到达 InfluxDB。

### 查看日志
```bash
# 查看 Telegraf 日志 (查看是否有解析错误, --tail=80 显示最后80行)
docker logs fsae_telegraf --tail=80
```

### 验证 MQTT 接收
```bash
# 订阅 MQTT 主题，查看原始数据是否到达 (仅显示前 10 条避免刷屏)
# -v 显示 topic
docker exec -it fsae_mosquitto mosquitto_sub -t fsae/telemetry -v | head
```

### 验证 InfluxDB 存储
```bash
# 查看数据库中 "telemetry" 表的所有字段 (验证 schema 是否正确)
docker exec -it fsae_influxdb influx -database fsae_db -execute 'SHOW FIELD KEYS FROM "telemetry"'

# 查询 "bms_data" 表的前 5 条数据
docker exec -it fsae_influxdb influx -database fsae_db -execute 'SELECT * FROM "bms_data" LIMIT 5'

# 带条件的查询：查询 module_id 为 2 的数据
# 注意：shell 中嵌套引号需要转义，这里单引号内部使用了 '\'' 来转义单引号
docker exec -it fsae_influxdb influx -database fsae_db -execute 'SELECT * FROM "bms_data" WHERE "module_id" = '\''2'\'' LIMIT 5'
```


## 3. 数据库数据清理

当修改了 Protobuf 定义或数据结构不对应时，通常需要清理旧数据。

### 清除特定表 (Measurement)
如果只修改了某个表及其关联的 Proto 里的 Message，可以只删那个表。

```bash
# 1. 停止写入 (可选，或直接执行删除)
# 2. 删除 telemetry 表
docker exec -it fsae_influxdb influx -database fsae_db -execute 'DROP MEASUREMENT "telemetry"'

# 3. 删除 BMS 数据表
docker exec -it fsae_influxdb influx -database fsae_db -execute 'DROP MEASUREMENT "bms_data"'

# 4. 删除后通常需要重启 Telegraf 以便重新建立连接或应用新配置
cd /home/admin/fsae_project
sudo docker compose restart telegraf
```

### 彻底重置数据库 (删库跑路风险，慎用)
如果数据完全乱了，想从头开始，可以直接删除挂载的文件夹。

```bash
# 1. 先停止服务
sudo docker compose down

# 2. 删除 InfluxDB 的数据挂载目录 (本地 Docker 卷)
sudo rm -rf influxdb_1.8_data

# 3. 重新创建空目录（Docker 启动时通常会自动创建，但手动建更稳妥）
mkdir influxdb_1.8_data

# 4. 重新启动服务
sudo docker compose up -d
```


## 4. Web 发布与生效

当前线上不是 Docker 里的 `fsae_web` 容器，也不是这个仓库直接承载 Web 源码。

线上实际方式是：

- Web 应用由宿主机 `pm2` 托管
- Nginx 在宿主机反代到 `127.0.0.1:3000`

因此 Web 发布后的常用命令应为：

```bash
# 重载或重启 Web 进程
pm2 reload bitfsae
pm2 restart bitfsae
pm2 save

# 修改 Nginx 配置后测试并重载
sudo nginx -t
sudo systemctl reload nginx

# 本机检查 Web 进程是否监听正常
curl -I http://127.0.0.1:3000
```

如果以后切回“纯静态站 + Nginx 直接托管”方案，再单独补对应命令，不要继续使用 `docker exec fsae_web ...` 这种旧方案命令。

## 6. 系统监控

在服务器（Linux 系统）中查看磁盘容量。

1. 查看整体磁盘剩余

```Bash
df -h
```
- **`-h` (human-readable)：** 将字节数转换为我们容易阅读的 GB、MB 等单位。
- 输出内容：
    - **Filesystem：** 硬盘分区的名称（如 `/dev/sda1`）。
    - **Size：** 该分区的总容量。
    - **Used：** 已使用的空间。
    - **Avail：** **剩余可用空间**（这是您最需要关注的）。
    - **Use%：** 已用百分比。
    - **Mounted on：** 挂载点（例如 `/` 是根目录，`/home` 是用户目录）。

2. 查看特定目录的大小
如果发现磁盘快满了，想知道是哪个文件夹占用的，可以使用 `du` 命令：

``` Bash
du -sh /path/to/directory
```
- **`-s` (summarize)：** 只显示总计，不列出子文件夹的明细。
- **`-h`：** 同样是以 GB/MB 格式显示。
- **常用技巧：** 查看当前目录下所有文件夹的大小并排序：
    ```Bash
    du -h --max-depth=1 | sort -hr
    ```

 3. 查看内存剩余情况

```Bash
free -h
```
- **total：** 总内存。
- **used：** 已用内存。
- **free：** 完全未使用的内存。
- **available：** **实际可用的内存**（包含可回收的缓存，比 free 更具参考价值）。

 4. 总结与对比

| **需求**             | **推荐命令**    | **说明**                   |
| -------------------- | --------------- | -------------------------- |
| **看硬盘剩多少**     | `df -h`         | 快速查看各个分区的占用情况 |
| **看某个文件夹多大** | `du -sh [路径]` | 定位哪些文件占用了大量空间 |
| **看内存剩多少**     | `free -h`       | 查看服务器当前的运行负荷   |

## 7. 配置修改速查表

修改不同文件后，需要执行不同操作才能生效：

| **修改对象** | **修改路径** | **生效方式** |
| :--- | :--- | :--- |
| **Web 应用代码** | 不在本仓库内 | 运行 `pm2 reload bitfsae` 或对应 Web 仓库的发布命令 |
| **Nginx 路由、SSL 证书配置** | 宿主机 `/etc/nginx/sites-available/bitfsae` | 运行 `sudo nginx -t && sudo systemctl reload nginx` |
| **Telegraf 采集规则** | `./telegraf/telegraf.conf` | 运行 `docker compose restart telegraf` |
| **Protobuf 定义** | `./protos/` | 运行 `docker compose restart telegraf`，并确保 client 同时更新 |
| **Grafana 面板配置** | 在线界面修改，持久化到 `./grafana_data` | 通常无需额外操作 |
