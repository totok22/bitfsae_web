# 服务端 Protobuf / Telegraf 核心说明

## 1. 当前服务端链路

1. 发送端把 Protobuf 二进制发到 MQTT
2. Mosquitto 接收消息
3. Telegraf 订阅 `fsae/telemetry` 和 `fsae/bms`
4. Telegraf 按 [`protos/fsae_telemetry.proto`](../protos/fsae_telemetry.proto) 解码
5. Telegraf 按 [`telegraf/telegraf.conf`](../telegraf/telegraf.conf) 里的映射提取字段
6. 数据写入 InfluxDB
7. Grafana 查询 InfluxDB

关键点：

- 服务端不是“改了 `.proto` 就自动适配”
- 真正决定入库结果的是 [`protos/fsae_telemetry.proto`](../protos/fsae_telemetry.proto) + [`telegraf/telegraf.conf`](../telegraf/telegraf.conf)
- 改协议后，服务端最核心的动作是同步修改 [`telegraf/telegraf.conf`](../telegraf/telegraf.conf) 并重启 Telegraf

## 2. 当前源码逐段说明

### 4.1 [`protos/fsae_telemetry.options`](../protos/fsae_telemetry.options)

当前源码：

```text
fsae.TelemetryFrame.modules    max_count:6
```

- 协议调整时，服务端文档里必须同时记录它，因为改 `modules` 数量时，服务端和 STM32 会联动

### 4.2 [`protos/fsae_telemetry.proto`](../protos/fsae_telemetry.proto)

```protobuf
syntax = "proto3";                // 使用 proto3 语法
package fsae;                      // Telegraf 中的类型名会写成 fsae.TelemetryFrame

message BatteryModule {

    uint32 module_id = 1;          // 模组编号，服务端当前把它作为 bms_data 的 tag

    // 23 节电芯电压
    uint32 v01 = 2;
    uint32 v02 = 3;
    uint32 v03 = 4;
    uint32 v04 = 5;
    uint32 v05 = 6;
    uint32 v06 = 7;
    uint32 v07 = 8;
    uint32 v08 = 9;
    uint32 v09 = 10;
    uint32 v10 = 11;
    uint32 v11 = 12;
    uint32 v12 = 13;
    uint32 v13 = 14;
    uint32 v14 = 15;
    uint32 v15 = 16;
    uint32 v16 = 17;
    uint32 v17 = 18;
    uint32 v18 = 19;
    uint32 v19 = 20;
    uint32 v20 = 21;
    uint32 v21 = 22;
    uint32 v22 = 23;
    uint32 v23 = 24;

    // 8 个温度
    sint32 t1 = 30;
    sint32 t2 = 31;
    sint32 t3 = 32;
    sint32 t4 = 33;
    sint32 t5 = 34;
    sint32 t6 = 35;
    sint32 t7 = 36;
    sint32 t8 = 37;
}

message TelemetryFrame {
    // --- 1. 基础信息 ---
    uint32 timestamp_ms = 1;       // 时间戳
    uint32 frame_id = 2;           // 帧序号

    // --- 2. 驾驶员输入 ---
    float apps_position = 3;       // 油门
    float brake_pressure = 4;      // 刹车压力
    float steering_angle = 5;      // 转向角

    // --- 3. 高压系统 ---
    float hv_voltage = 6;          // 高压电压
    float hv_current = 7;          // 高压电流
    float battery_temp_max = 8;    // 电池最高温
    uint32 fault_code = 9;         // 故障码

    // --- 4. 动力系统 ---
    int32 motor_rpm = 10;          // 电机转速
    float motor_temp = 11;         // 电机温度
    float inverter_temp = 12;      // 逆变器温度

    // --- 5. 车辆状态 ---
    uint32 ready_to_drive = 13;    // Ready To Drive 状态
    uint32 vcu_status = 14;        // VCU 状态

    // --- 6. BMS 详细数据 ---
    repeated BatteryModule modules = 15; // 电池模组数组
}
```

服务端角度的重点：

- [`telegraf/telegraf.conf`](../telegraf/telegraf.conf) 中 `xpath_protobuf_type = "fsae.TelemetryFrame"`，顶层类型就是这里的 `TelemetryFrame`
- `fsae/bms` 并不是另一个独立 message，仍然是 `TelemetryFrame`，只是服务端只提取其中的 `modules`
- 改字段名后，Telegraf 对应 XPath 要同步改
- 改字段类型后，Telegraf 里的 `fields` / `fields_int` 要同步改
- 不要使用bool类型（可能会有问题），此外，要注意服务器带宽和STM32发送带宽有限的问题。

### 4.3 [`telegraf/telegraf.conf`](../telegraf/telegraf.conf)

```toml
[agent]
  interval = "100ms"                    # 采集周期
  round_interval = true
  metric_batch_size = 1000
  metric_buffer_limit = 10000
  collection_jitter = "0s"
  flush_interval = "1s"                # 每 1 秒刷写一次
  flush_jitter = "0s"
  hostname = "fsae-server"

[[inputs.mqtt_consumer]]
  servers = ["tcp://mosquitto:1883"]   # MQTT Broker 地址，容器内走服务名 mosquitto
  topics  = ["fsae/telemetry"]         # 第一条链路：基础遥测

  data_format = "xpath_protobuf"       # 使用 Protobuf + XPath 解码
  xpath_protobuf_import_paths = ["/etc/telegraf/protos"]
  xpath_protobuf_file = "fsae_telemetry.proto"    # 容器内读取挂载进去的 proto 文件
  xpath_protobuf_type = "fsae.TelemetryFrame"     # 顶层 message 类型

  [[inputs.mqtt_consumer.xpath]]
    metric_name = "'telemetry'"        # InfluxDB measurement 名

    [inputs.mqtt_consumer.xpath.fields_int]
      timestamp_ms = "//timestamp_ms"  # 顶层整数字段
      frame_id     = "//frame_id"
      fault_code   = "//fault_code"
      motor_rpm    = "//motor_rpm"
      ready_to_drive = "//ready_to_drive"
      vcu_status     = "//vcu_status"

    [inputs.mqtt_consumer.xpath.fields]
      apps_position    = "number(//apps_position)"    # 顶层浮点字段
      brake_pressure   = "number(//brake_pressure)"
      steering_angle   = "number(//steering_angle)"
      hv_voltage       = "number(//hv_voltage)"
      hv_current       = "number(//hv_current)"
      battery_temp_max = "number(//battery_temp_max)"
      motor_temp       = "number(//motor_temp)"
      inverter_temp    = "number(//inverter_temp)"

[[inputs.mqtt_consumer]]
  servers = ["tcp://mosquitto:1883"]   # 仍然从同一个 MQTT Broker 订阅
  topics = ["fsae/bms"]                # 第二条链路：BMS 数据

  data_format = "xpath_protobuf"
  xpath_protobuf_file = "fsae_telemetry.proto"
  xpath_protobuf_import_paths = ["/etc/telegraf/protos"]
  xpath_protobuf_type = "fsae.TelemetryFrame"       # 仍按 TelemetryFrame 解码

  [[inputs.mqtt_consumer.xpath]]
    metric_name = "'bms_data'"         # InfluxDB measurement 名
    metric_selection = "//modules"     # 遍历 TelemetryFrame.modules，每个 module 生成一条记录

    [inputs.mqtt_consumer.xpath.tags]
      module_id = "module_id"          # module_id 作为 tag

    [inputs.mqtt_consumer.xpath.fields_int]
      v_01 = "v01"                     # 当前把所有电芯电压和温度都映射成整数字段
      v_02 = "v02"
      v_03 = "v03"
      v_04 = "v04"
      v_05 = "v05"
      v_06 = "v06"
      v_07 = "v07"
      v_08 = "v08"
      v_09 = "v09"
      v_10 = "v10"
      v_11 = "v11"
      v_12 = "v12"
      v_13 = "v13"
      v_14 = "v14"
      v_15 = "v15"
      v_16 = "v16"
      v_17 = "v17"
      v_18 = "v18"
      v_19 = "v19"
      v_20 = "v20"
      v_21 = "v21"
      v_22 = "v22"
      v_23 = "v23"

      t_1 = "t1"
      t_2 = "t2"
      t_3 = "t3"
      t_4 = "t4"
      t_5 = "t5"
      t_6 = "t6"
      t_7 = "t7"
      t_8 = "t8"

[[outputs.influxdb]]
  urls = ["http://influxdb:8086"]      # 输出到 InfluxDB 容器
  database = "fsae_db"
  skip_database_creation = false
```


### 3.1 服务端命令

在仓库根目录执行：

```bash
docker compose restart telegraf
```

用途：

- 重载 [`telegraf/telegraf.conf`](../telegraf/telegraf.conf)
- 重载挂载到容器里的 [`protos`](../protos/) 目录

状态检查：

```bash
docker compose ps
docker compose logs telegraf
```

### 3.2 Python 模拟命令

在 [`protobuf_doc`](./) 目录执行：

```bash
protoc --proto_path=../protos --python_out=. ../protos/fsae_telemetry.proto
```

### 3.3 STM32 / Nanopb 命令

确保下载了 [Nanopb](https://jpa.kapsi.fi/nanopb/) (通常是 `nanopb-0.x.x-windows.zip`)，解压后将其 `generator-bin` 目录加入 PATH，或者直接调用其中的 `nanopb_generator`。

在 [`protos`](../protos/) 目录执行：

```bash
cd protos
nanopb_generator fsae_telemetry.proto
```

或：

```bash
cd protos
python path/to/nanopb/generator/nanopb_generator.py fsae_telemetry.proto
```

执行成功后，会得到：

*   `fsae_telemetry.pb.c`
*   `fsae_telemetry.pb.h`

**将这两个文件复制到 STM32 工程中，替换旧文件。**
