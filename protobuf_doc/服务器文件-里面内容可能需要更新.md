# 服务器文件说明

这份文档只保留当前仓库里仍然有效的服务器侧配置说明。

当前应以仓库根目录下这些文件为准：

```text
docker-compose.yml
telegraf/telegraf.conf
protos/fsae_telemetry.proto
protos/fsae_telemetry.options
```

## 1. 当前线上部署基线

当前线上不是 Docker 里的 `fsae_web` 容器，也不是仓库内静态站直出。

现网实际结构是：

- 宿主机 `nginx` 负责 `80/443`
- 宿主机 `pm2` 负责 Web 进程 `bitfsae`
- `docker compose` 只运行 `mosquitto`、`influxdb`、`telegraf`、`grafana`
- Grafana 通过 `127.0.0.1:3001` 暴露，再由 Nginx 反代到 `/monitor/`

因此，不要再把旧的 `web:` 容器段当成现网配置。

## 2. 当前 `docker-compose.yml` 要点

当前 compose 的有效服务只有 4 个：

```yaml
services:
  mosquitto:
    image: eclipse-mosquitto
    container_name: fsae_mosquitto
    ports:
      - "1883:1883"
      - "9001:9001"
    volumes:
      - ./mosquitto/config/mosquitto.conf:/mosquitto/config/mosquitto.conf
      - ./mosquitto/data:/mosquitto/data
      - ./mosquitto/log:/mosquitto/log
    restart: always

  influxdb:
    image: influxdb:1.8
    container_name: fsae_influxdb
    ports:
      - "8086:8086"
    volumes:
      - ./influxdb_1.8_data:/var/lib/influxdb
    environment:
      - INFLUXDB_DB=fsae_db
    restart: always

  telegraf:
    image: telegraf
    container_name: fsae_telegraf
    volumes:
      - ./telegraf/telegraf.conf:/etc/telegraf/telegraf.conf:ro
      - ./protos:/etc/telegraf/protos:ro
    depends_on:
      - influxdb
      - mosquitto
    restart: always

  grafana:
    image: grafana/grafana
    container_name: fsae_grafana
    ports:
      - "127.0.0.1:3001:3000"
    environment:
      - GF_DASHBOARDS_MIN_REFRESH_INTERVAL=200ms
      - GF_SERVER_DOMAIN=bitfsae.xin
      - GF_SERVER_ROOT_URL=https://bitfsae.xin/monitor/
      - GF_SERVER_SERVE_FROM_SUB_PATH=true
    volumes:
      - ./grafana_data:/var/lib/grafana
    restart: always
```

如果将来迁移到新域名，只改 Grafana 这两项即可：

- `GF_SERVER_DOMAIN`
- `GF_SERVER_ROOT_URL`

## 3. `telegraf.conf` 修改规则

Telegraf 使用 `xpath_protobuf` 解析 Protobuf 数据。

只要改了下面任一内容，就必须同步检查 `telegraf/telegraf.conf`：

- `protos/fsae_telemetry.proto` 中的字段名
- message 结构
- measurement 映射关系

否则最直接的结果就是：MQTT 能收到，InfluxDB 没数据，或者字段名错位。

### 当前实际是两段独立映射

当前 `telegraf/telegraf.conf` 里有两个 `[[inputs.mqtt_consumer]]`：

1. `topics = ["fsae/telemetry"]`
2. `topics = ["fsae/bms"]`

它们虽然都解析 `fsae.TelemetryFrame`，但用途不同：

- 第一段把基础遥测写成 measurement `telemetry`
- 第二段通过 `metric_selection = "//modules"` 把每个电池模组写成 measurement `bms_data`

因此修改协议时要分清楚：

- 改 `TelemetryFrame` 的基础字段，检查第一段
- 改 `BatteryModule` 的字段，检查第二段

如果只改了其中一段，另一段不会自动同步。

### 字段类型和 Telegraf 配置要对应

当前配置里：

- 整数字段放在 `fields_int`
- 浮点字段放在 `fields`

例如：

- `motor_rpm` 在 `fields_int`
- `hv_voltage` 在 `fields`

所以如果你把一个字段从 `uint32` 改成 `float`，不仅要改 `.proto`，还要把它在 Telegraf 中从 `fields_int` 挪到 `fields`。

### 修改后如何让配置生效

在仓库根目录执行：

```bash
docker compose restart telegraf
```

如果是首次确认配置问题，也可以看容器状态和日志：

```bash
docker compose ps
docker compose logs telegraf
```

## 4. 哪些文件不在这个仓库里

下面这些属于现网 Web 运行环境，但不在本仓库：

- `/opt/bitfsae/.output/`
- `/opt/bitfsae/ecosystem.config.cjs`
- `/opt/bitfsae/.env`
- `/etc/nginx/sites-available/bitfsae`
